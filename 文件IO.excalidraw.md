1.创建文件空洞

2.获取文件大小
    处理文件前判断、限制文件大小

       大文件分片处理（断点续传）、配置文件按段读取
4.修改覆盖指定内容
    修改配置文件、更新日志文件状态标记 ^QRYu6xpn

三类核心功能
1.复制文件描述符(重定向标准输出)
    程序日志重定向、守护进程将输出写入文件（避免终端依赖）
2.获取/修改文件描述符的属性
    程序运行中动态调整文件的读写规则（比如日志文件从覆盖变成追加）
    设置文件为非阻塞模式
    适用场景:网络编程（socket读写）、多进程/线程处理文件(避免阻塞整个程序)
3.实现文件锁（配合struct flock），解决多进程竞态
    多个进程同时读写一个文件时，会出现竞态问题。struct flock
定义锁的规则，fcntl执行加锁/解锁
    排他锁（写锁）：同一时间只有一个进程能写，读也被阻塞
    共享锁（读锁）：多个进程能同时读，写会被阻塞

struct flock {
    short l_type;    // 锁类型：F_RDLCK（共享读锁）、F_WRLCK（排他写锁）、F_UNLCK（解锁）
    short l_whence;  // 锁的偏移基准（同lseek：SEEK_SET/SEEK_CUR/SEEK_END）
    off_t l_start;   // 锁的起始偏移量
    off_t l_len;     // 锁的长度（0表示到文件末尾）
    pid_t l_pid;     // 持有锁的进程ID（仅F_GETLK时有效）
}; ^fv1RDr5S
